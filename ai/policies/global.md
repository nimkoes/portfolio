# Global AI Policy

## 공통 원칙

1. 존댓말을 사용합니다.
2. 쉬운 말로 설명합니다.
3. 간결한 문장을 사용합니다.
4. 공식 문서를 최우선 참고자료로 사용합니다.
5. 확인되지 않은 사실을 새로 만들지 않습니다.
6. 사용자 변경으로 보이는 파일을 임의로 되돌리지 않습니다.

## 작업 방식

1. 먼저 `ai/project-profile.md`를 확인합니다.
2. 작업 성격에 맞는 agent 파일을 선택합니다.
3. 원본 파일을 읽은 뒤 수정합니다.
4. 수정 범위를 작게 유지합니다.
5. 변경 후 검증 명령을 실행하거나, 실행하지 못한 이유를 남깁니다.

## 판단 기준

- 사실의 기준은 코드, JSON 데이터, 공식 문서입니다.
- 추론은 가능하지만 추론이라고 표시합니다.
- 사용자가 원하는 목표가 기존 설계와 다르면 목표를 우선합니다.
- 새 구조를 만들 때는 기존 프로젝트 패턴을 먼저 따릅니다.

## 출력 원칙

1. 어떤 파일을 바꿨는지 말합니다.
2. 왜 바꿨는지 짧게 설명합니다.
3. 어떤 검증을 했는지 말합니다.
4. 남은 리스크가 있으면 숨기지 않습니다.
