# Quality Policy

## QA 기본 원칙

- 빌드 성공만으로 완료로 보지 않습니다.
- 데이터, 화면, 접근성, 배포 경로를 함께 확인합니다.
- 발견한 문제는 재현 조건과 기대 결과를 함께 기록합니다.

## 필수 점검

1. `npm run lint`
2. `npm run build`
3. 콘텐츠 변경 시 `npm run ai:loop`
4. 모바일 레이아웃
5. 라이트/다크 테마
6. 접힘/펼침 버튼
7. 외부 링크

## 접근성 기준

- 버튼은 `aria-label`이나 명확한 텍스트를 가져야 합니다.
- 펼침 영역은 `aria-expanded`, `aria-controls` 관계를 유지해야 합니다.
- 장식용 요소는 `aria-hidden`을 유지합니다.
- 키보드 포커스가 사라지지 않아야 합니다.
